Assign each user a portal role. Changes take effect on next request (middleware uses Django is_staff / is_superuser for admin access, and UserProfile.role for faculty vs student dashboards).

What each role can do

  • Student — Student dashboard, submissions, enrollments; cannot access faculty course tools or Django admin.
  • Instructor — Faculty dashboard, create/manage courses and assignments, rosters; cannot access Django admin by default.
  • Admin — Full Django admin and GradeSync admin console (is_staff). Superuser accounts are labeled separately and can only be changed by another superuser.
